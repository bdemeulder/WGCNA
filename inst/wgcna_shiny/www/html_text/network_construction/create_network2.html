Detect Modules

If the user chooses the Automatic construction method, module detection uses default values for the needed parameters.

If the user chooses the Manual construction method, he/she will have access to the following parameters:

  • Min module size (by default 30 genes): minimum number of genes in a module,
  • Max joining height (by default 0.995): genes above this threshold in the Cluster Dendrogram are put in the grey module (~garbage module),
  • Depth of split (by default 2): depth of branch allowed for the module detection (from 0 to 4).

The parameter values will influence the way WGCNA defines the initial set of gene modules. The user can click on the Define Initial Modules button to launch the computation.

The modules can then be iteratively merged (by clicking on the Merge Close Module button) if they are similar enough (i.e. below the merge threshold). The red line in the "Clustering of module eigengenes" plot shows roughly which modules are going to be merged (the grey module is never merged). As the merging works iteratively, decreasing the Threshold to Merge Modules between 2 iterations will have no effect.

For the manual computation, the modules are defined initially using the default parameters but their values can be changed and the modules will have to be re-initialised by clicking on the Define Initial Modules button.
